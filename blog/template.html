inline vs inline-block

Display property

To simplify your comprehension on this subject I suggest that you try to visualize all the elements as boxes. That's basically how the elements are treated on a webpage.

So, in this article I am gonna explore the CSS display property, which is commonly used to define how the elements will be placed in your page. It is a very important topic, specially if you wanna learn how to create better layouts because it allows you to control the behavior of those elements.

There are different values for this CSS property. But today we are gonna focus on two of them: display inline and inline-block.

Inline

First let's understand the main difference between display: inline and display: block.

The display: inline, as the name implies, place the items side by side and it causes no line break. The attributes height and width do not apply to these elements because it can only fill their own content width.

Now, it is important to highlight that the box model works in a different way in this case. It can accept attributes such as, padding and margins but it will only be applied horizontally (left and right).

Examples of inline elements: anchor, span, image and also formatting tags, such as, strong and italic.

Block

Now on display: block the elements are placed vertically by default, so in this case, it does cause a line break. These elements takes 100% of the parent element's width and it doesn't matter the content you put in there, it will always occupy all the space available.

Examples of block elements: div, paragraph, heading, section, footer, etc...

Inline-block

The display: inline-block looks like an inline element from the outside, but behaves like a block element. It can accept paddings and margins in all sides and it is commonly used to create navigation links and buttons.

Inline

  • Padding and margin only works horizontally
  • Causes no line breaks
  • Occupy only the content's space
  • Heights and widths do not apply

Inline-Block

  • Similar to an inline element
  • Causes no line breaks
  • Occupy only the content's space
  • Box-model can be applied

Block

  • The elements are display as blocks
  • Placed vertically (causes line breaks)
  • Occupy 100% of the parent's width
  • Box-model can be applied
